Design decisions

Choices made where the spec was ambiguous.

decision2026-06-16 Implementation-notes file lives at repo root and persists (only the exec plan is deleted on the final commit). It mirrors the plan's task checklist and the Decision Log.
decision2026-06-16 · Phase A The inversion lives in a new module scripts/vf_invert.py, called late in build-variable-font.py (after lilx/ss13/long-arrows, before the mono-rebase). It edits only cmap + GSUB (no new glyphs), so repair_hvar is not needed after it. Categories: parens + 12 arrows → cmap default; a NEW default-on calt carries Recursive dlig ligatures + long arrows + forward ss03/06/08/10/11 + the lilx connected/thin lookups; lilx/ssNN/ss13 rebuilt as reverts.
decision2026-06-16 · Phase A Code ligatures are now default-on in the VF (registered under the new calt, which HarfBuzz/CoreText apply by default). This makes the full Moxy look — including connected dashes/bars and long arrows — render with zero ghostty config, matching the static build (which already uses dlig2calt). The original dlig feature is kept too (harmless; lets a user still request it explicitly).

Deviations

Intentional departures from the spec, and why.

deviation2026-06-16 · Phase A Letter reverts are MONO=1-tuned. Recursive's forward ss maps are many-to-one (f and f.mono both → f.simple), so a single-substitution revert can't be MONO-aware. The revert restores the .mono form (correct at Moxy's canonical MONO=1 / Mono default). At MONO=0 (Sans), enabling ssNN/ss13 yields f.mono/r.mono instead of the sans f/r. Accepted: Moxy lives at MONO=1; the Sans-revert edge case is cosmetic.
deviation2026-06-16 · Phase A “lilx+ss13 = pristine Recursive” holds to within <1 unit. The residual sub-unit delta is the pre-existing mono-default-rebase rounding (handoff: “±1 unit”), not a Phase A change — the inversion never touches glyf/gvar. The 12 arrows + long-arrow fix are additive/always-on, so the reverted font is “Recursive + arrows”.

Tradeoffs

Alternatives considered and why the chosen path won.

tradeoff2026-06-16 · Phase A cmap remap vs. default-on feature for the letterforms: a cmap remap to f.simple would fight Recursive's rvrn MONO swap (f → f.mono). Putting the forward ss lookups in a default-on calt instead composes cleanly with rvrn at every MONO value (verified at MONO 0/1). cmap remap is still used for parens/arrows, which rvrn does not touch.
tradeoff2026-06-16 · Phase A Reverse via contextual re-analysis vs. unconditional single-subs. The Moxy connected/thin glyphs (*.lilx, *.seq) only ever appear because the default calt produced them, so reverting them unconditionally (plus one Type-4 religature for \X escapes) is both simpler and exactly correct — no need to re-run the original contextual chains in reverse.
